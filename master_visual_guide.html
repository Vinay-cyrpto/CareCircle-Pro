🗺️ Total Controller & Request Map

Comprehensive technical overview of every controller, who triggers it, and its core action.

🔑 Auth Service (Identity Layer)

Controller Who Calls? Request / Path Core Action
AuthController All Users POST /auth/** Login, Registration, OTP verification, and JWT issuance.

👤 User Profile Service (User Metadata)

Controller Who Calls? Request / Path Core Action
ParentProfileController Parent POST/GET /parents/profile Creation and management of parent bio and address.
CaregiverController Caregiver POST/PUT /caregiver/** Management of skills, experience, and bio.
ChildController Parent POST/GET /children/** Managing child profiles linked to parent account.
AdminController Admin GET/PUT /admin/** Moderation and profile approval workflows.
CommonLookupController All GET /common/** Fetching static lookup data (Genders, Types).

📅 Matching & Booking Service (Business Logic)

Controller Who Calls? Request / Path Core Action
MatchingController Parent POST /search Elite Search Engine: Finding caregivers by city & time.
BookingController Parent POST /bookings Creation of a booking with Overlap Check validation.
StatusController Caregiver PUT /bookings/{id}/status Lifecycle: Moving booking from REQUESTED to ACCEPTED.
AvailabilityController Caregiver POST /availability Setting the 'Ready to Work' time slots.
RateController Parent POST /bookings/rate Review system: Updating caregiver's cumulative rating.

💬 Communication Service (Messaging)

Controller Who Calls? Request / Path Core Action
ChatController All POST /chats/rooms Initializing "Safe Space" chat tied to a valid booking.
WebSocketController System PUB /topic/chat/{id} Real-time STOMP broadcasting of messages.
BlockController All POST /blocks Safety mechanism to prevent communication with a user.
Technical Pattern: All requests are authenticated at the Gateway (8080) and arrive at controllers with pre-validated X-User-Id and X-User-Role headers.
